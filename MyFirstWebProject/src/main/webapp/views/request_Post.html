POST방식으로 요청 후 응답데이터 받아오기

특징. 1. POST방식으로 요청을 보내면 BODY영역에 데이터를 포함시켜서 요청
(편지지에 쓰는 것) => 사용자가 입력한 값(데이터)들이 URL에 노출되지 않음
=> 보안유지가 가능함
=> 로그인, 회원가입 같은 기능은 POST방식이 적합함
2. 전송하는 길이에 제약이 없음
(사실 길이 제약 있는데 길면 나눠서 보냄.) => 즉, 게시글 작성 같은 경우 POST방식이 적합함
헤더는 길이가 정해져있음(32byte정도) 이 길이를 넘어갈 수 없다
POST는 바디에 적으므로 이 제약을 받지 않는 것
3. 최대 요청 받는 시간이 존재함
(GET방식은 하염없이 기다림)
요청이 너무 많이 들어오면 요청 거부함

4. 새롭게 데이터를 만들어내는 기능에 적합하다. CRUD중에 C의 Insert에 적합

간단한 인적사항 - POST
